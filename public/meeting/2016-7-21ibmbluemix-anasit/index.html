一、企业信息化的市场前景

企业信息化技术的发展

  • 2000-2008
    • 自建机房
    • IDC托管
    • 门户网站、OA、ERP
  • 2009 各大公司陆续发布公有IaaS服务
  • 2012 云计算时代的PaaS元年
    • Hadoop、MapReduce被IBM、亚马逊、谷歌、和微软等财力雄厚的PaaS提供商封装成预制服务
  • 2015
    • 全面融入IT领域的容器技术元年
    • Windows Server 2016将为Docker类容器技术提供支持
    • OpenStack、CloudFoundry等Linux巨头，也在重新对其产品线中的某些核心组件进行调整
    • 2015年中国企业级SaaS市场规模将达到199.3亿元，这是一个正经历着裂变式高速增长的市场
  • 2016 变革从此爆发
    • 预计2016年中国企业级SaaS市场规模将超过300亿元人民币
    • 混合云 IBM Bluemix 中国区将正式发布

中小企业信息化的需求

  • 互联网技术革新速度不断加速，二三线城市还未跟上步伐
  • 企业信息系统，该选择私有云还是公有云？
    • 如果公有云，该选择IBM、AWS还是国内服务商？
    • 如果私有云，该选择私有云IDC托管还是自建机房？
    • 企业云资源虚拟化，该选择OpenStack还是Docker？

这些头疼无比的选择，正反映了中小型企业信息化的现状：信息不对等，导致了中小型企业难接触飞速发展的企业信息化技术。

二、消灭信息不对等！

消灭常见认知误区

  1. 十年前，建设一个门户就是企业信息化
  2. 现在，做一个公众号，发布一款APP就是企业信息化
  3. 只要有WEB，拼凑几个智能硬件，就是互联网+项目

消灭成本预估误区

  1. 公司策划的项目很好，只差一个程序员
    • 对系统需求不明确，导致资金预算与预期成果严重不符
  2. 请一个软件外包团队，几个月就能做完项目
    • 对系统上线运维成本不了解，导致项目难以持续迭代

案例1：江西某旅游集团信息化转型

关键工具

IBM API Connect

  • 安全地打通现有的信息数据
  • 可视化API构建集成
  • Swagger2.0规范标准
  • API使用统计分析

项目背景

2015.4月，该旅游集团互联网+旅游产品与2015年1月开始已上线一个季度，由一家小型非技术型公司来外包完成产品开发。平台前期设计开发过于仓促，上线运营缺少维护支持以及全方位测试。导致了其正式上线运营后劲不足，影响到集团线下战略发展。

危机分析

  • 拥有200家子公司，但集团内部信息化系统杂乱
    • 新员工入职需要先学习五种系统，日志汇报系统、财务审批系统、旅游产品系统、客户管理系统、微信商城系统
  • 各子系统的交接环节，完全使用人工手动统计与交接
  • 线上产品经测试后漏洞较多，导致关键业务需要重构

提供支持

  • 重构微平台关键业务——旅游产品商城，对接财务报表系统
  • 对接集团内部信息系统，部分打通API，部分子系统直接重构
  • 优化线上产品自动化部署及自动化运维策略
  • 设计平台全套RESTful APIs，提供信息化数据微服务

成果

  1. 大幅度减少集团原有的人力资源浪费（信息化及自动化）
  2. 利于项目维护及后续扩展开发（基于RESTful重构后的线上产品数据接口）
  3. 为该集团提供技术岗位招聘需求，及产品迭代运维方案

二三线城市软件危机，主要原因归纳

  • 服务方：小型作坊过多，中小型软件外包服务公司不规范
    • 人才难以寻求，二三线城市互联网技术人才紧缺。
    • 成本难以压缩，小型公司外包业务利润率，仅有30%。
    • 领头人非技术出身，技术策略偏重重复生产，而非技术革新
  • 企业方：采购SaaS服务接口开放度不高，或企业内无人进行信息整合
    • SaaS服务兴起，但多为通用模板，定制化较少
    • 缺少合适的技术人才，进行信息整合优化
    • 中小企业（非互联网领域）信息技术投入多为负支出，所带来成果难以见效

三、中小企业如何顺利进行信息化转型？

1.混合云对中小企业快速实现信息化转型的影响

采用敏捷实践——改变中小型企业构建软件的传统方式

有人评论说：”针对目前国内情况，DevOps还是很遥远。也许只有行业顶尖的公司，或者新成立的公司会有这样的尝试。大多数的企业还未开始进行敏捷的推进，传统的重重阻碍会使敏捷的推进进程遥遥无期。” DevOps真的离我们有那么远吗？DevOps应该从哪里开始呢？

  • 持续部署和交付

利用专为缩减时间和降低管理运营成本而设计的平台，缩短软件交付周期。 - DevOps

营造一个充满信赖、文化强大的结构性平台去服务企业连接需求和实施，事半功倍。

云上的DevOps——选用合适的混合云集成方案

  • 原型测试阶段
    • 公有云——IBM Bluemix Public
    • 适用于：小型规模应用，或测试期产品
  • 公测迭代阶段
    • 私有托管——IBM Bluemix Dedicated
    • 适用于：用户规模较大，注重数据独立性与可靠性，需要硬件独立的私有环境
  • 大规模发布阶段
    • 私有部署——IBM Bluemix Local
    • 适用于：公司有专业运维部门，对IaaS或PaaS层有一定的定制需求

3.二三线城市软件服务将重新洗牌

服务方与企业方

    • 新技术创业公司的冲击
