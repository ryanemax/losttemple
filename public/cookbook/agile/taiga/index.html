0.当敏捷团队碰到Agile系统

0.0 尝试

之前尝试过很多优秀的Agile工具：

1.odoo 一体化的公司开源神器

优点：全面的ERP，自带项目模块支持Kanban;

缺点：缺少APP以及移动端，用户体验不足

总结：现阶段团队需要更加专注敏捷开发工作，而且功能简洁易用的辅助系统，而全面的Odoo不适合现阶段，以后公司规模扩长，它会是一个集CMS、ERP、CRM、财务、库管一体的优秀系统。

2.Zentao 支持国产

优点：国人开源的项目管理系统，中文文档丰富

缺点：插件收费，手机版收费，界面体验不足，集成功能只适合做Scrum

0.1 选择

0.1.1 敏捷开发及项目管理系统

核心必备：

  • Kanban 任务完成情况最直接的表示方式，把黑白纸条搬到线上
  • Issue 用于产品测试及系统上线环节必备的排错维护
  • Wiki 建立API文档、开发参考文档、计划书等团队经验分享
  • BackLog 一定量的判断开发的工作效率、以便控制适当的开发周期

因此，选择了Taiga，它是集以上关键功能一体，简洁，美观，开源，免费的敏捷管理系统。

0.1.2 即时通讯视频

taiga-front 编译安装方式

Taiga项目的前端系统页面，AngularJs开发，支持扩展Modules、Themes

一般直接使用taiga-front-dist

ruby 2.1(only for sass)

nodejs 0.10(for npm,gulp,bower,coffescript)

ruby/sass

$ gem install sass scss-lint
$ export PATH="~/.gem/ruby/2.1.0/bin:$PATH"
$ sass -v // should return Sass 3.3.8 (Maptastic Maple)

Node + Bower + Gulp

$ sudo npm install -g gulp
$ sudo npm install -g bower
$ npm install
$ bower install
$ gulp

taiga-front-dist git安装方式 git clone https://github.com/taigaio/taiga-front-dist cp -r taiga-front-dist/dist /var/www/taiga

taiga-back

python 3.4+

postgresql >= 9.3
