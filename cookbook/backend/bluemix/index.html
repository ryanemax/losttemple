Mobile

总评：弱势但不乏亮点

Bluemix作为优秀API整合平台，在移动端应用构建方面，略显弱势，暂时无法广泛应用于生产环境。

部分消息推送、用户分析、统计服务，可尝试性接入产品进行试用，以便于对接Bluemix大数据服务项目。

消息推送服务：Push Notifications

结论 - 推荐系数：☆☆☆☆ - 价格实惠、客户端SDK完善 - 略优于市面上混杂的推送服务

费用参考

  • 每月前一百万条数字消息（推送通知）免费
  • US$ 1.05 USD/百万条数字消息
项目 PN 极光 Parse
开源 否 否 是
推送方式 SDK、APNs、GCM、Chrome、Firefox SDK、APNs、GCM APNs、GCM
广播推送 是 是 是
标签分组 是 是 需开发
并发 未测试 未测试 自主扩容

数据CURD服务：Mobile Application Content Manager

结论 - 推荐系数：☆ - 功能贫瘠，无法替代firebase、parse.com等专业MBaaS服务 - 缺少权限控制，需要自主编写逻辑 - 数据库无法独立部署，后期拓展限制较大

功能

  • Cate
    • 类似Mongodb的Collections，用于存放数据对象的集合
  • Content
    • 数据对象设计，可自定义数据对象成员属性及字段

前端UI（APP）快速构建工具：Kinetise

结论 - 推荐系数：☆ - 成熟度，无法替代Ionic Creator及APICloud等前端快速构建平台 - 可作为初学练习，快速成型Demo的构建 - 可导出项目源码，后续优化可自行编程
